# 阶段 0：市场情报扫描深度细则（v0.1.1）

## 为什么必做

短剧市场每月洗牌。今天爆的"老年女性觉醒"半年前还没有，今天烂大街的"赘婿逆袭"两年前是天花板。

不知道当下热什么 → 写出来就是一年前的烂梗。

v0.1.0 的 market-pulse **只搜标签**（"老年觉醒 红果"），导致 agent 凭训练数据写桥段，写出来的"双胞胎调换+下毒+装糊涂律师"是 2010 年套路。

v0.1.1 强制 **WebFetch 真读爆款拆解文章**，提取**具体桥段**而非标签。

---

## 4 子步流程（必做）

### 0.1 联网搜当下 Top 5 剧名

```
搜索 1：红果短剧 排行榜 <YYYY-MM>
搜索 2：番茄小说 改编 IP <近 30 天>
搜索 3：抖音短剧 <用户题材关键词> 热度
搜索 4：DataEye 短剧观察 / QuestMobile 月报 <YYYY-MM>
搜索 5：短剧 平台 审核 <当前年> 红线
```

提取：

- 近 30 天 Top 5 剧名
- 每部的集数 / 单集时长 / 热度数据
- 用户题材在不在 Top 5 / 是否撞车

### 0.2 真读 1-2 部爆款的业界拆解（核心）

不要只看标签。**WebFetch** 1-2 部最相关爆款的剧情解说 / 业界拆解文章：

```
首选源（按可访问性排序）：
1. woshipm（人人都是产品经理）— 多数 200 OK，硬数据多
2. 36 氪 / 21 经济网 / 钛媒体 — 多数 200 OK，趋势分析
3. 腾讯新闻 news.qq.com — 多数 200 OK
4. 搜狐 sohu.com — 多数 200 OK
5. 知乎 zhuanlan.zhihu.com — 403 反爬概率高，备用
6. B 站文字稿 / 公众号 — 视情况
```

WebFetch 提取：

- 开场怎么写（前 3 秒 / 前 30 秒）
- 主线节奏（爆点频率、反转间隔）
- 反转点设置（一卡 / 二卡 / 终极反转）
- 爽点动作的具体形态（不是"打脸"，而是"用什么方式打脸"）
- 角色辨识度技巧（标志动作 / 口头禅 / 视觉母题）
- 失败案例 / 被吐槽点

如果所有 WebFetch 都被反爬挡住 → fallback 到内置 `references/red-fruit-data.md` 的硬数据库 + 案例库。

- [蓝海 / 红海前夜 / 红海 / 衰退] — 因为 <X>

## 内置硬数据保底

每次启动 skill 时，agent 都应**先读** `references/red-fruit-data.md`（内置 woshipm + 业界硬数据 + 案例库）。

WebFetch 只是为了**更新**这个内置数据，而不是替代它。

如果 WebFetch 全部失败 → 直接用内置数据，但要告诉用户「本次未联网更新，使用内置数据（最后更新：YYYY-MM）」。

## 反同质化检测

用户题材如果撞近 30 天爆款（≥ 3 部），必须做：

1. 提炼 3 部爆款的共同套路（开场怎么写 / 男女主关系 / 反转点）
2. 列出用户题材**不能这样**写的部分
3. 给 2-3 个**完全不同**的切入角度

例：用户说"我要做奶奶觉醒题材"

**Top 3 爆款共同套路**：
- 奶奶发现儿媳虐待（《奶奶觉醒后全家破产了》）
- 奶奶手撕儿媳分家（《奶奶您继续》）
- 太奶奶穿越降临（《十八岁太奶奶》）

**不能这样写**：
- 不能再用「全家围攻奶奶 → 奶奶反击」开场（已被洗烂）
- 不能再用"突然觉醒"作触发点
- 不能再用"奶奶是隐藏律师 / 神医"作反差

**差异化角度**：
- A. 奶奶不是觉醒，是装糊涂 30 年早就觉醒了
- B. 觉醒的不是奶奶，是孙女，奶奶是幕后军师
- C. 奶奶觉醒后不复仇，转而救一个外人（陌生女孩），震慑全家

差异化是本技能区别于通用编剧的关键。

## 与 v0.1.0 的核心差别

| 项 | v0.1.0 | v0.1.1 |
|----|--------|--------|
| 信息源 | 仅搜标签 | **真 WebFetch 业界拆解** + 内置硬数据 + 标签搜 |
| 出报告 | 一次性写完整报告 | 4 子步互动 |
| 用户介入时机 | 写完才看 | **0.3 选项菜单**让用户挑差异化角度 |
| 内置数据 | 无 | `red-fruit-data.md` 含案例库 + 月度更新机制 |
