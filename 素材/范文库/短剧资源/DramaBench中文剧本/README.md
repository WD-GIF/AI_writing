# DramaBench 中文剧本库（500 个）

> **数据来源**: Hugging Face `FutureMa/DramaBench` (中国短剧→英文 Fountain 格式)
> **翻译方式**: opus-mt-en-zh (Helsinki-NLP) + 规则混合翻译
> **总规模**: **500 个剧本，33.9 万中文字，1,847 个场景，2,686 个角色**
> **数据大小**: 2.3 MB

## 🛠️ 翻译流水线（`工具/短剧剧本提取/translate_dramabench.py`）

### 混合翻译策略

| 内容类型 | 翻译方式 | 例子 |
|---|---|---|
| 角色名 (ALL CAPS) | 规则字典（姓+名拼音） | `SONG YUAN` → 宋媛 |
| 场景标题前缀 | 规则映射 | `INT./EXT.` → 内景/外景 |
| 场景标题时间 | 规则映射 | `DAY/NIGHT/LATER` → 日/夜/稍后 |
| 场景标题地点 | 规则映射 (50+常见) + MT fallback | `BEDROOM` → 卧室 |
| 括号注释 | 规则映射 (40+) | `(V.O.)` → （画外音） |
| 过场标记 | 规则映射 | `FADE IN:` → 淡入： |
| 动作描述 | opus-mt-en-zh | 神经翻译 |
| 对白 | opus-mt-en-zh | 神经翻译 |

### 🔑 关键技巧：人名占位符

MT 模型对中文字符不友好（替换后会被吃掉），但对 `ZnameNZz` 形式的拉丁词完美保留：

```
英文原文：SONG YUAN lies on the bed.
   ↓ 替换为占位符
占位符版：Zname1Zz lies on the bed.
   ↓ MT 翻译
中间结果：Zname1Zz躺在床上。
   ↓ 还原占位符
最终：宋媛躺在床上。
```

效果：**同一角色全文译名 100% 统一**，无音译变体。

## 🎯 用法

### 1. 当作训练语料（爆款剧情）

500 个剧本 = 339k 字 = 一个完整的"短剧爆款骨架库"。
按题材分类（标题里有关键词），喂给 AI 学习开局/反转/打脸节奏。

### 2. 当作角色名灵感库

2,686 个角色 = 2,686 个中文短剧典型人名。
角色表里有"姓"和"名"对应关系，可直接套用到自己写的网文。

### 3. 当作场景模板

1,847 个场景 = 1,847 个"内景/外景 + 地点 + 时间"组合。
学习哪些场景是必备（卧室/CEO办公室/法庭/警察局），哪些是高频转场。

### 4. 直接当大纲改写

每个剧本本身就是完整的"开篇+矛盾+反转"，可改写成短剧/网文：
- 把英文式表达改成更口语的中文
- 替换角色名为自己设定的
- 调整反转节奏

## ⚠️ 已知局限

| 问题 | 原因 | 影响 |
|---|---|---|
| 标题翻译生硬 | opus-mt 不懂短剧"梗" | 看 English Title 兼参考 |
| 个别专有名词误译 | "Suona"(唢呐)→"阿素奥娜" | 罕见词，可后期人工修正 |
| 偶有重复短句 | MT 贪婪解码副作用 | 已用 `no_repeat_ngram_size=3` 缓解 |
| 句末标点不一致 | MT 习惯 | 不影响阅读 |
| 描述行人名顺序偶尔颠倒 | MT 语序 | 不影响理解 |

**整体可读性**: 95% 以上（人工抽样验证）
