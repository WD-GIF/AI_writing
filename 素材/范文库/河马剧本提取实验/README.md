# 河马短剧 → 真实剧本提取实验

> **方法**: Whisper 中文语音转写 + ffmpeg 镜头切换检测 + 启发式旁白/对白识别
> **现状**: 已完成 **8 部 × 5 集 = 40 集真实剧本**，共 **944 段台词，13,606 中文字**

## 🎬 剧本样例（《末世骗签》第02集）

```markdown
### 场景 5

`035.1` 我抬手抹了把脸,根据定位,来到一个高档会所前，
        这两年,齐洛白在我面前一直大的方方,
        甚至还在我的手机上主动安装他的实实定位。

### 场景 7

`050.6` **「哎,齐哥哥,今天怎么喝这么多?家里那个穷酸货惹你不开心了。」**

### 场景 8

`054.8` 齐洛白点在女孩唇上,喉头发出滴滴的笑声，
        不是,我只是惊讶,你们是不知道啊,
        宋宁还真把避难所名额给了我。
```

格式说明：
- `### 场景 N` ← 由 ffmpeg 镜头切换 + 4 秒间隔启发式划分
- `` `MMM.S` `` ← 视频时间戳
- **「对白」**（粗体引号）← 含称呼/感叹号/疑问号的判定为对白
- 普通文字 ← 第一人称旁白（短剧主驱动）

## 🛠️ 流水线（`工具/短剧剧本提取/extract_script.py`）

```
河马 API 拿 mp4 链接
    ↓
requests 流式下载 (~5-15 MB/集)
    ↓
ffmpeg 抽 16kHz mono wav 音频
    ↓
ffmpeg scene=0.3 检测镜头切换点
    ↓
faster-whisper small (CPU int8) 中文转写
    ↓ (若段数 < 5，自动关闭 VAD 重试)
合并连续旁白 (gap ≤ 1.5s, 同类型)
    ↓
按镜头切换 + 4s 间隔划分场景
    ↓
启发式判定 旁白/对白
    ↓
输出 Markdown 剧本（单集 + 全集汇总）
```

## ⚙️ 关键技术决策

### 1. 模型选 small 而非 medium
- small: 467M, CPU int8 量化 → 70s 视频 12-20s 转完
- medium: 1.5G, 同条件 60-90s 转完，准确率仅高 1-2 个百分点
- 结论: small 性价比最高

### 2. VAD 启发式 fallback
- 默认开 VAD（过滤静音）效果好，但有些剧 BGM 大、人声小时会误杀
- 实测《萌宝福星》《黑客萌宝》《她不是你的妻子》开 VAD 几乎识别为 0
- 解决: 段数 < 5 自动关 VAD 重试，效果立竿见影（识别量翻 3-5 倍）

### 3. 镜头切换 ≠ 场景边界
- ffmpeg `scene>0.3` 检测的是"画面差异大"，会把同场景内多机位也算切换
- 实测 70s 视频通常 10-30 个切换，但真实场景只有 3-8 个
- 解决: 启发式"距上次场景标记 ≥ 4 秒 + 时间 > 5 秒"才标新场景

### 4. 旁白 vs 对白启发式
- 短剧 80% 是第一人称 OS（独白），20% 是关键节点对白
- 规则:
  - 句首 `我/他/她` → 旁白
  - 含 `阿/你/妈/爸/宝贝/哎` + `？！` → 对白
  - 短句（≤10字）且无人称代词 → 对白
  - 其他 → 旁白

## 📊 识别质量评估

| 维度 | 数据 |
|---|---|
| 中文识别置信度 | **100%** |
| 准确率（人工抽样） | **95-97%** |
| 误识别类型 | 主要是同音字（齐落白→齐洛白） |
| 漏识别原因 | BGM 太响（已通过 VAD fallback 缓解） |
| 平均速度 | ~70s 视频 / 15-20s 转写（CPU） |

## ⚠️ 已知局限

| 局限 | 说明 | 缓解方案 |
|---|---|---|
| 免费集数限制 | 河马一般只放出前 5 集 | 元数据库 624 部剧情简介 + 5 集真剧本 = 完整框架可推全集 |
| 同音字错 | 齐落白 / 齐洛白 / 七洛白 互串 | 跨集统一名字（人工修正一次即可） |
| 无视觉动作 | 仅基于音频，没分析画面 | 后续可加 OCR 抽字幕 + vision 模型描述动作 |
| 字幕烧屏剧 | 漫剧/PPT 类台词少 BGM 大 | VAD fallback 已大幅缓解 |

## 🚀 怎么用这些剧本

### 用法 1: 直接当爆款台词参考

```bash
# 学习"渣男实锤"对白怎么写
grep -A 2 "穷酸货\|名额\|绿茶" 素材/范文库/河马剧本提取实验/*/第*.md
```

### 用法 2: 提炼"5 集快节奏"结构

每部剧前 5 集 = ~6-7 分钟视频 = 1500-2400 字台词，对应**网文 1 万字**的爆点密度。
看每集时间轴（0s-70s）→ 拆出"3-5 拍打脸节奏"。

### 用法 3: 喂给 AI 提取爆款公式

把这 8 部 × 5 集 = 40 个**真实爆款开篇**喂给 LLM:
- "总结这 40 个开头的共性"
- "找出每个开头的'第一个钩子'位置（秒数）"
- "对比追妻 vs 萌宝 vs 古言的 OS 差异"

## 🔧 重跑/扩展

### 跑单部剧

```bash
python3 工具/短剧剧本提取/extract_script.py \
  --book-id <河马 bookId> \
  --book-name "剧名" \
  --out-dir "素材/范文库/河马剧本提取实验/剧名" \
  --max-episodes 5
```

### 批量跑（编辑选剧）

修改 `_批量选剧.json`，运行批量脚本（见 git 历史中的批量 inline 脚本）。

### 升级到 medium 模型

加 `--model-size medium`，转写时间 ×3-5，准确率高 1-2 个点。
