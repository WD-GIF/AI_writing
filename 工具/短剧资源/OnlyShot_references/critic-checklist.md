## 6 维度评分细则

### 1. 黄金 3 秒钩子强度

测试方法：把第 1 集前 3 秒读给一个不知本剧的人听，看 ta 是否想看下去。

- 10：立刻问"然后呢"
- 8-9：感兴趣
- 6-7：将信将疑
- ≤ 5：会划走

打分标准：
- 有具体动作 / 冲突 / 反转 → +3
- 第 1 句话就是宣战 / 揭秘 / 极端动作 → +3
- 视觉冲击强（撞 / 砸 / 摔 / 血）→ +2
- 钩子是熟梗（套路化）→ -3
- 钩子需要 5s 以上才能 get → -5

woshipm 数据：捉奸 / 甩脸 / 耳光 / 砸杯子 类强刺激开场，完播率最高。

### 2. 可预测度

测试方法：看完前 3 集，能猜中结局吗？

- 10：完全猜不到，但回头看伏笔都对
- 8-9：能猜大方向，细节意外
- 6-7：能猜大半
- ≤ 5：第 1 集就猜到了 → 致命问题

如果 ≤ 5：必须改。改的方向：
- 加一个反派的反派（让真正的幕后黑手不是观众第一直觉的那个）
- 男女主关系反转（从仇人变盟友 / 从盟友变仇人）
- 主角身份反转（最后揭穿其实是 X）

### 3. 同质化风险

测试方法：列出近 30 天同题材爆款的 3 部，逐项对比。

每和一部爆款相似一项 -1 分。

如果 ≤ 5：要么改差异化角度，要么挖一个独有的「视觉母题 + 角色反差」让用户记住。

### 4. 爽点密度

测试方法：数全集 satisfaction 数量，按 woshipm 数据标准：

| 集长 | 最低爽点数 | 满分爽点数 |
|------|-----------|------------|
| 60s | 2 | 4 |
| 120s | 4 | 8 |
| 180s | **6** | **12** |

打分：每少 1 个 -1 分。

新增：每分钟必须 ≥ 3 个情绪爆点（small_climax / twist / satisfaction），不达标扣 -2 分。

### 5. 角色辨识度（5 特写人格锚定）

测试方法：把所有角色名换 A/B/C，让人读对话，能猜出谁是谁吗？

每个角色必须有：
- 5 特写人格锚定（v0.1.1 新增）→ +3
- 独特口头禅 → +1
- 独特动作 → +1
- 独特视觉母题 → +1
- 鲜明性格反差 → +2

满分 8。低于 5 → 角色没立住，回去重写人物 bible。

### 6. 老套度（v0.1.1 新增）

由 5.2 反套路检测决定：

- 一级黑名单命中：每处 -3 分
- 二级黑名单命中：每处 -1 分
- 三级黑名单命中：警告，不扣

总扣分从 10 分基础扣，最低 0 分。

判定：
- 9-10：原创度高
- 6-8：常见组合但有反差
- 3-5：套路堆叠（必改）
- 0-2：纯套路（结构性返工）

### 7. 付费点强度（仅付费短剧）

测试方法：第 8、26、50、72 集结尾 paywall_punch 反转，强到能让人掏钱吗？

- 10：必须知道下一集，立刻付费
- 8-9：很想知道，犹豫一下会付费
- 6-7：好奇但可能不付费
- ≤ 5：致命，付费率扑街

打分标准：
- 是核心反转（不是过渡）→ +3
- 涉及主角身份 / 关系 / 命运 → +3
- 兑现前 N 集所有伏笔 → +2
- 留 cliffhanger 半句话 → +2

## 修订态度

- **不要赞美**自己写的（"这个钩子很好" 都不要写）
- **不要"中立"**（"可以更好" 等于没说）
- **要尖锐**：直接写"这个钩子是套路，会被划走"
- **要有具体改写方案**：不要说"建议加强"，要说"把第 1 句改成 'XXX'"

如果你写的批判全是 8-10 分 → 你不敢真批判 → 回去用更狠的眼光再看一遍。

## 修订流程（v0.1.1 新增归档机制）

### 大改触发

- 老套度 ≤ 5 → 流派可能选错或多处必改
- 同质化风险 ≤ 5 → 题材撞车
- 6 维度任意 3+ 维度 ≤ 5 → 结构性返工

大改前**自动归档**：

```bash
mkdir -p 归档/v1.0_2026-05-05/
cp -r ./*  归档/v1.0_2026-05-05/  (排除 归档/ 自身)
```

更新 `项目元数据.json` 的 `revision_count`、`version`。

### 小改

- 单维度 ≤ 5 但其他维度 ≥ 7 → 仅局部修订
- 单条对白改 / 单镜改 → 直接改

写入 `09_修订记录.md`：

```markdown
## YYYY-MM-DD vX.Y（<大改/小改>）
- 改了 EP01 第 3 镜对白：「你疯了」→「您消消气」（让张兰更装哭）
- 影响：分集/第01集_断亲/{剧本.md, 分镜.json, 镜头清单.md, 即梦批量包.md}
- 触发原因：critique 6 维度评分 5/10，必改 #2
- 评分变化：钩子 8 → 9
```

## 用户介入时机

写完 `08_自我批判.md` 后给用户看，让用户：

- 确认修订优先级
- 决定哪些建议采纳、哪些不采纳
- 提出额外修订点

然后**真的去改**——不是写完批判就结束，要在原文件上落地修订。

修订完后再做一次小批判，分数有没有上来。如果某项还是 ≤ 5，继续改。
