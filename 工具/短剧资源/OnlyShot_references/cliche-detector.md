# 反套路检测器（v0.1.1 新增）

每次创作完阶段 1（IP 简报）和阶段 4（前 3 集脚本），都要对照本黑名单做 grep 检测。命中 → 警告 → 给替代方案。

阶段 5 自我批判时把检测结果写入「老套度」维度。

## 三级黑名单（轻度老套，慎用）

- 寿宴 / 婚礼 / 葬礼当众撕脸
- 录音笔藏家里证据
- DNA 报告作为关键证据
- 监控录像反转
- 银行流水截图
- 微信聊天记录截图

这些不是不能用，但**一集只能用 1 个**，多用 = 套路化。

## 检测流程（skill 在阶段 5 自动执行）

```python
# 伪代码
def detect_cliche(scripts):
    cliche_list = parse(this_file)
    hits = []
    for cliche in cliche_list:
        if any(keyword in scripts for keyword in cliche.keywords):
            hits.append({
                "cliche": cliche.name,
                "level": cliche.level,
                "files": match_files,
                "replacement": cliche.replacement
            })
    return hits
```

实际运行时由 LLM 读黑名单 + 通读全部剧本文件 + 输出命中清单。

## 维护机制

本黑名单**每季度更新**。新晋"被洗烂"的套路（如某 2026.5 起被滥用的桥段）要加入。

跟踪信号：
- 知乎"短剧吐槽"标签
- 豆瓣短剧讨论
- B 站短剧解说视频弹幕"又是这个"

---

## 例外情况

下列情况可以**保留**老套桥段（即使命中黑名单）：

1. **故意致敬**（如年代爽剧故意复刻 90 年代套路）
2. **反套路使用**（先让观众以为是套路，再反转打脸）
3. **多重叠加创新**（如「双胞胎调换 + 退休大法官 + 装糊涂 30 年」组合在 v0.1.0 算"老套堆叠"，但如果改成「单胎被换 + 退休税务官 + 高调装愚 5 年」+ 反派也是双胞胎反向操控，就算反套路使用）

但例外要在 critique 里明确说明 + 用户确认。
