# 阶段 1：IP 定位深度细则（v0.1.1）

## 5 子步互动流程

跟 v0.1.0 最大不同：**不要一次写完整 IP 简报让用户看完**。每子步给选项菜单让用户挑。

### 1.1 流派选择（关键，决定后续写法）

参考 `references/genre-flavors.md`。给用户 5 选项：

```markdown
## 你想做哪种流派？（决定后续写作风格）

A. 红果纯爽流（推荐红果用户首选）
   - 生活流冲突 + 当集即爽
   - 节奏：每分钟 4 个爆点
   - 代表：《奶奶觉醒后全家破产了》《有本事别复婚》
   - 适合你的题材吗？<判断>

B. 精品悬疑流
   - 长线伏笔 + 30 年家族秘密
   - 节奏：每分钟 3 个爆点
   - 代表：《女相师》《云渺》
   - v0.1.0 默认走这个，看 24 集才揭晓 → 红果观众不耐烦

C. 漫剧奇观流（2026 红果新赛道）
   - AI 漫剧 / 系统 / 末世 / 神话
   - 节奏：每分钟 5 个奇观点
   - 代表：《菩提临世真人 AI 版》
   - 注意：AIGC 标识强制 + 审核更严

D. 沙雕轻喜流
   - 段子密度高
   - 代表：《好一个乖乖女》

E. 年代爽剧流
   - 80-90 年代背景 + 重生 / 穿越
   - 代表：《我在 80 年代当后妈》

选 1 个。流派选错后面全错。
```

**等用户回复**再进 1.2。

### 1.2 题材原型（1 主 + 1 副）

读 `references/archetype-catalog.md`。给用户 5 个**主原型**选项 + 3 个**副原型**选项：

```markdown
## 主原型（按你的流派和市场情报推荐）

A. 老年觉醒（女频核心）
B. 重生复仇
C. 神医 / 相师 / 命格
D. 离婚追妻火葬场
E. 重生 + 商战

挑 1 个主原型。

## 副原型（给主原型加调味）

a. 家族秘史
b. 法律智斗
c. 商战 / 经商
d. 玄学 / 命理
e. 反派群像

挑 1 个副原型。
```

### 1.3 核心钩子（草拟 3 版让用户挑）

```markdown
## 核心钩子（≤ 25 字）

基于你选的「<流派> + <主原型> + <副原型>」，我草拟 3 版钩子：

A. <版本 A：聚焦动作冲击>
B. <版本 B：聚焦悬念问句>
C. <版本 C：聚焦反差画面>

挑 1 个，或告诉我想改的方向。
```

**钩子要求**：
- ≤ 25 字
- 看到立刻想知道"然后呢"
- 自带矛盾或悬念
- 不能是设定，必须是动作 / 反转 / 冲突

❌ 反例：
- "讲一个穿越当皇后的故事"（这是设定，没钩子）
- "女主很厉害，男主追她"（没具体冲突）

✅ 正例：
- "女主穿越落地砸晕皇帝，发现自己是先皇宠妃"
- "奶奶在 80 大寿当天宣布全家断亲，理由 30 年没说出口"
- "重生后她做的第一件事，是给前世的'闺蜜'写讣告"

### 1.4 重复爽点公式（草拟 2 版让用户挑）

公式结构：`<触发条件> + <主角动作> + <对方反应>`

```markdown
## 重复爽点公式（每集都要复现的"上瘾点"）

A. 触发：<X> + 主角动作：<Y> + 对方反应：<Z>
   预期出现频率：每集 N 次

B. 触发：<X'> + 主角动作：<Y'> + 对方反应：<Z'>

挑 1 个，或自己改。
```

例（红果纯爽流）：
- A. 触发：婆婆当众挤兑女主 → 主角动作：女主用一句话点破婆婆藏的心事 → 反应：婆婆失态、转移话题、装病
- B. 触发：邻居议论女主丈夫出轨 → 主角动作：女主翻出丈夫工资条+保险单 → 反应：邻居哑口、丈夫尴尬

例（精品悬疑流）：
- A. 触发：全家逼奶奶让步 → 主角动作：奶奶慢悠悠从老物件抽出证据 → 反应：先愣→辩解→垮掉

### 1.5 写入 `02_IP简报.md`

整合 1.1-1.4 + 受众 + 对标 IP + 集数定价 + 视觉关键词，按模板写文件。

文件结构（用 `assets/templates/02_IP简报.md` 模板）：

- frontmatter
- 市场快照（从 01_市场情报.md 引用关键摘要）
- IP 7 维度
- 用户确认清单（每项打勾）

## 7 维度细则

### 1. 题材原型

- 主原型 + 副原型
- 原型组合解释（这个组合预期带来什么差异化）

### 2. 目标受众

| 维度 | 选项 | 怎么选 |
|------|------|--------|
| 性别 | 女频 / 男频 / 全龄 | 红果 81.7% 女频 |
| 年龄段 | 18-25 学生 / 25-40 都市 / 40+ 中老年 | 老年觉醒 → 40+ |
| 平台 | 红果 / 抖音 / 快手 / 番茄 / TikTok | 决定时长和审核 |
| 付费意愿 | 免费 / 付费 | 决定卡点结构 |

### 3. 对标 IP（"X 的设定 + Y 的节奏"）

至少 2 部已爆短剧。说明对标的具体维度。

### 4. 核心钩子

来自 1.3 用户选定。

### 5. 重复爽点公式

来自 1.4 用户选定。

### 6. 集数与定价

| 模式 | 集数 | 单集 | 付费点 |
|------|------|------|--------|
| 免费短剧 | 70（默认）| 180s | 无（广告分成）|
| 付费短剧 | 70-100 | 180s | 第 8-12 集首付费、26-30 集二付费、50-55 集三付费 |
| 漫剧 | 50-70 | 180s | 同付费短剧 |

### 7. 基调与视觉

| 基调 | 视觉关键词 |
|------|----------|
| 沙雕爽 | `vibrant colors, sitcom lighting, exaggerated expressions` |
| 狗血爽 | `high contrast, dramatic lighting, melodrama, telephoto compression` |
| 治愈甜 | `soft warm light, shallow depth of field, pastel palette` |
| 暗黑虐 | `low key lighting, desaturated, chiaroscuro` |
| 玄学神秘 | `misty, dim ambient, candlelight, mystical symbols` |

视觉风格关键词每段 prompt 都要带（即梦每段独立生成，不带就跳风格）。
