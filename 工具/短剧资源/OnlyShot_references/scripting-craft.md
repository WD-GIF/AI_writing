# 阶段 4A：逐集脚本写作细则（v0.1.1）

适配 **180s × 60 grid** 的新参数。

---

## 单集结构（180s 通用）

```
[0-3s]      冷开场（cold_open）— grid 1
[3-30s]     第一段冲突展开 — grid 2-10
[30-60s]    第一爆破（耳光级）— grid 11-20
[60-120s]   推进 + 反转 — grid 21-40
[120-150s]  第二爆破 — grid 41-50
[150-170s]  卡点铺陈 — grid 51-57
[170-180s]  cliffhanger — grid 58-60
```

每分钟 ~20 grid，每分钟必须 ≥ 3 个情绪爆点。

## 每 grid 的脚本格式

```markdown
### grid X（time_start - time_end）

**画面**：<具体画面，含人物动作、表情、运镜>
**对白**：
> <角色>[语气]：「<台词 ≤ 15 字>」 [节奏标记]
**潜台词**：<这句话表面是 X，潜台词是 Y>
**音效 / BGM**：<具体音效配方，如：钢琴单音 + 80bpm 弦乐持续>
```

---

## 对话原则

按 woshipm + red-fruit-data：

1. **每句 ≤ 15 字**
2. **不要解释，靠动作**（"她生气了" → 她把茶杯摔了）
3. **每句话都有目的**（推进剧情 / 揭示性格 / 制造冲突，三选一）
4. **潜台词 > 表面意思**（最好的台词表面平静，底下刀光剑影）
5. **不要内心独白**（短剧没空间，靠表情演）
6. **冲突双方对话不超过 3 个回合**（再多就拖；要么有第三人插入，要么镜头切走）
7. **每分钟 ≤ 60 字对白**（红果观众消化能力上限）

---

## 节奏标记（在脚本中用方括号标）

| 标记 | 含义 |
|------|------|
| `[钩子]` | 这句话是钩子 |
| `[爽点]` | 这句话/动作触发爽点公式 |
| `[反转]` | 这里是 twist |
| `[虐点]` | 这里是 pain |
| `[卡点]` | 这里是 cliffhanger |
| `[伏笔]` | 这里埋 setup（注明 payoff_ep）|
| `[兑现]` | 这里兑现伏笔（注明 setup_ep）|
| `[母题]` | 视觉 / 听觉母题出现 |

---

## BGM / 音效写法

不要写"激烈音乐"——给配乐师 / 即梦音效模型没用。

写**具体音效配方**：

| 情绪 | 写法 |
|------|------|
| 爽点 | 鼓点猛击 + 反向音效 + 高频 hi-hat 渐入 |
| 虐点 | 钢琴单音 + 低频弦乐持续 + 80bpm |
| 钩子 | 心跳声 + 突然静音 + 脚步声 |
| 反转 | 唱片刮擦音效 + 0.5s 静音 + 鼓点 |
| 揭秘 | 弦乐渐强 + 风铃 + 镜头长焦 |
| 30 秒爆破 | 耳光声 + 玻璃碎裂音效 + 鼓点 + 唱片刮擦 |

## 单集自查

- [ ] 0-3s 让陌生观众想看下去？
- [ ] 30s 有第一个爆破点？
- [ ] 全集对话总字数 ≤ 180×1=180 字（每分钟 ≤ 60 字 × 3 分钟）
- [ ] 每句台词 ≤ 15 字
- [ ] 没有内心独白
- [ ] 节奏标记齐全
- [ ] BGM 写到具体音效配方
- [ ] 卡点是 5 种万能模板之一
- [ ] 反套路检查无命中

---

## 分集 scope 控制（默认）

72 集 × 60 grid × 4 文件 = 16800 文件单元，全做不现实。

**默认套餐**（用户可改）：

A. 第 1、8、27、72 集（4 集，最关键节点）
B. 第 1-5 集（开篇 5 集）
C. 第 1、8、27、50、72 集（5 集节点）
D. 全 72 集（贵且慢，用户确认才做）

阶段 4.1 给用户挑套餐。中间集只在 `03_完整剧本.md` 写大纲 + `04_节奏地图.json` 写 beat。
