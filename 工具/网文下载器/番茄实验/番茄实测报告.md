# 番茄小说爬虫实测报告

> **结论先行**：技术上跑得通，但**字体加密只能解 38.8%**，剩下 60% 是乱码。**不推荐爬番茄**。

---

## 实测过程

### 步骤 1：直接访问
- `fanqienovel.com` 可达 ✅
- 书籍详情页能拿到 ✅
- 章节正文页能拿到 4488 字符的"文本" ✅

### 步骤 2：发现字体加密
- 抓回来的文本包含 **4502 个 PUA 私有区字符** (U+E000–U+F8FF)
- HTML 引用了自定义字体 `DNMrHsV173Pd4pgy`
- 浏览器渲染时通过 woff2 字体把 PUA 字符 → 真实汉字
- **直接抓 = 全是乱码**

### 步骤 3：尝试已有解密方案
- 用 `zhoulianglen/fanqiexiaoshuo-Download` 项目内置的 362 字映射表
- 实测：**解密率 38.8%**
- 中文字数 1420 / PUA 剩余 2244

---

## 为什么解密率这么低？

- 字体里有 **362 个 glyph**（包含 a-z, A-Z, 0-9 + 300 个汉字）
- 但**所有 248 个剩余 PUA 字符都在字体里**——它们有字形，只是项目映射表没收录
- 原因：**番茄定期更新字体**，glyph ID 编号变了，老映射表失效

---

## 解密样本

见 `解密样本-部族荣光-第1章.md`，前两段：

```
[?]乾皇朝，疆域亿[?][?]。
北[?]，黑潭[?]，七[?]初九。
统治周遭百岭[?][?][?][?]千[?][?]久[?]黑潭[?][?][?][?]族...
```

`[?]` 代表未解密字符。**完全没法读**。

---

## 完整解密的工程量

要做到 95%+ 解密率，需要：

1. **OCR 自动生成映射表**：
   - 渲染字体中每个 glyph 成图片
   - 用 PaddleOCR / EasyOCR 识别每张图
   - 生成 `glyph 字形 hash → 真实汉字` 映射
   - 工程量：**约 200-400 行代码 + 几百 MB OCR 模型**

2. **字形 hash 化**（更稳定）：
   - 用 glyph 坐标计算 hash
   - 即使番茄换字体也能复用
   - 工程量：更大，需要爬一遍字形数据库

3. **逆向 JS 找到 Token + API**：
   - 番茄有官方 API
   - 但需要登录 + 设备指纹
   - 工程量：**最大**，且违反 ToS 风险高

---

## 推荐路线

**🚫 不推荐**：现在就花时间搞番茄解密

**理由**：
- 用户目标是收集 **8000-12000 字短篇爽文**
- 番茄主要是**长篇**（10-100 万字）
- 即使爬下来也是几十万字的长篇，得截取分析
- 字体破解工程量大，回报不成正比

**✅ 推荐**：
1. 用户自己**手机微信里收集公众号文章 URL**——这才是 8-12k 字短篇的真正金矿
2. AI 这边继续用现有的 2 篇范文 + 11 张人物卡 + 6 张情节卡撑起第 1 个项目
3. 后续如果真要爬番茄，**单独开一个 1-2 周的项目**，专门做字体解密 + OCR
