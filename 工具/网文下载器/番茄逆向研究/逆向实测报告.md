## 💡 真正可行的方案

由于发现了 **`/api/reader/directory/detail` 这个 JSON API**，我们可以完整地批量爬番茄而**不需要 X-Argus**：

```
1. 用 fanqienovel.com/page/{book_id}    → 拿书籍基础信息
2. 用 /api/reader/directory/detail       → 拿完整章节列表（含付费状态）✅ NEW
3. 用 /reader/{item_id} HTML              → 抓章节正文
4. 用 OCR 自动生成字体密码本             → 解密 PUA 字符
5. 输出完整可读小说                       ✅
```

**这个流程是我们 `fanqie_decoder.py` 已经实现的，只是没用 API 抓章节列表**（用了 HTML 解析）。可以小升级一下。

| 步骤 | 工具 | 时间 |
|---|---|---|

| 3. Python 实现签名生成 | 改装 ssovit 项目 | 2-3 天 |

**总投入**：8-13 天 × 维护成本 = **几个月的工程**

**收益**：能调 App API 的所有接口（搜索 / 推荐 / 个性化等）

**对比**：现有方案（不需要签名）已经能下载完整内容了，**只是没有搜索而已**。

## 🛠️ 我现在能做的实际改进

### 改进 1：用 directory API 替代 HTML 解析（小升级）

```python
# 老方法：解析 HTML 找章节链接
for a in soup.find_all('a', href=re.compile(r'/reader/\d+')):
    ...

# 新方法：直接调 API（更稳，含付费标记）
r = requests.get(f"https://fanqienovel.com/api/reader/directory/detail?bookId={book_id}")
chapters = r.json()['data']['chapterListWithVolume'][0]
# 每章: {itemId, title, needPay, isPaidPublication}
```

好处：
- 一次请求拿到所有章节
- 准确识别付费章节（needPay = 1）
- 不依赖 HTML 结构变化

### 改进 2：批量下载多本书

现有 `fanqie_decoder.py` 已经支持单本。可以加 `--batch <书 ID 文件>` 选项。

### 改进 3：找书功能的替代方案
