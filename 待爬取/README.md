# 待爬取队列

> **用法**：你把要爬的 URL 列表丢到这个文件夹，跟我说"跑一下"，我就批量跑+push 到仓库。

---

## 📥 提交 URL 的两种方式

### 方式 A · GitHub 网页直接编辑（最方便）

1. 进入这个文件夹
2. 点 `urls.txt`（如果没有就新建一个）
3. 右上角 ✏️ 编辑
4. 一行一个 URL 粘贴进去
5. 滚到底 Commit changes
6. 跟我说："**跑一下 urls.txt**"

### 方式 B · 对话里直接贴

把 URL 列表直接粘到对话里，我会：
1. 帮你保存到 `urls-<日期>.txt`
2. 立刻跑
3. push 结果

---

## 📂 文件夹结构

```
待爬取/
├── README.md                    ← 本文件
├── urls.txt                     ← 主队列（一行一 URL）
├── urls-追妻火葬场.txt           ← 按题材分类（可选）
├── urls-重生复仇.txt
└── 已完成/                       ← 跑完后我会自动归档
    └── urls-2026-06-16.txt
```

---

## 📝 urls.txt 格式

```
# 一行一 URL，# 开头是注释会忽略
# 自动识别站点：公众号 / 笔趣阁 / 知乎 / 通用

# 追妻火葬场 - 公众号
https://mp.weixin.qq.com/s/xxxx

# 重生复仇 - 笔趣阁
https://www.biqu5200.com/book/xxxxx/

# 知乎盐选
https://zhuanlan.zhihu.com/p/xxxxx
```

## ⚙️ 跑爬虫时的默认设置

| 参数 | 默认值 | 说明 |
|---|---|---|
| 字数下限 | 8000 | 短篇爆文目标长度下限 |
| 字数上限 | 12000 | 短篇爆文目标长度上限 |
| 输出目录 | `素材/范文库/<日期>/` | 按日期归档 |
| 反爬延迟 | 1-2.5 秒 | 章节间随机 |

如果你想改默认设置，直接跟我说。

---

## ⚡ 我跑完会做这 4 件事

1. **下载到 `素材/范文库/<日期>/`**
2. **字数不符的移到 `_筛除/`**
3. **生成 `汇总.md`**：列出每篇标题、字数、来源
4. **git push 到 main 分支**

你 `git pull` 或在 GitHub 网页上就能看到。

---

## ⚠️ 注意

- 公众号链接**会过期**，提交之后越快跑越好
- 一批别超过 30 篇（VM 时间有限）
- 我跑的时候每篇随机延迟，全程大约 1-3 分钟一篇
- **不要交 VIP 付费内容** —— 跑了也是空白
